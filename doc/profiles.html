1. Introduction

The profile feature allows the user to change the Amaya interface.

The user can select the functions he wishes to have in the menus ane toolbar buttons and build his own profile.

2. How to use it

2.1. Creation of a profile

  • Create a file (named as you wish) in your personal Amaya configuration directory (~/.amaya under UNIX) and get it ready for edition**.
  • Open the file Amaya/config/ProfileDefs. This file defines all the functions available in Amaya. These functions are spread into modules (look at the file structure below). This file is useful to know which modules and functions are available.
  • Give the profile you're working on a title : Type this title between brackets.
  • Under that title, you can add as much modules or functions as you wish (look at An example of profile configuration file).
  • You can add as much profiles as you want in the same profile file but remember you'll be only proposed the 6 first profiles in the Amaya profile selection..

Note : **All these operations can be done with Amaya. You can open, edit and save your profile configuration even if the extension of the file isn't html.

2.2. Profile selection

A dialog box has been added in Amaya in order to handle profile selection.

  • Open this dialog box.

    Special -> Preferences -> Profile...

  • Fill in the field "Location of profiles definition file" with the path of the created profile file. This choice allows you to have severals profile definition file.
  • Restart Amaya
  • Reopen the profile dialog box
  • If everything' ok, you are proposed one or several profile**. Check the radio box matching the profile you want to use and click the Apply button.
  • Restart Amaya for the changes to take effect.

Note : ** The profiles choice is limited to 6 profiles. So, if you defined 10 profiles in the same file, don't be surprised if only the six first profiles are proposed.

3. An example of profile configuration file

This example shows two profiles definitions (Amaya4Kids and Editor_Novice between brackets).

All strings preceded by a # are modules (defined in ProfileDefs) and others are functions. Modules are units of functions.

[Amaya4Kids]

#editbase

#browse

#templates

CreateHeading1

CreateList

CreateImage

[Editor_Novice]

#editbase

#basicelements

#links

#templates

#views

4. Structure of the ProfileDefs file

The ProfileDefs file contain units of modules definition separated by comments lines beginning with several # (like ####VIEWS). These comments are there to help you searching your module (or function).

Beginning of module definition is represented by a the name of module between brackets (like [views] or [zoom])

All the modules or functions under this starting module element belong to the module as long as another starting module element starts.

######### VIEWS

[views]

#zoom

#docviews

TtcSwitchButtonBar

TtcSwitchCommands

ShowMapAreas

ShowTargets

wmwShowToC

HelpViews

TtcCloseView

[docviews]

ShowStructure

ShowAlternate

ShowLinks

[zoom]

ZoomIn

ZoomOut
