About Amaya’s Accessibility Features

Accessibility for people with disabilities is being improved with each release. Currently, you can perform most Amaya functions without using a mouse, and carry out most editing functions (except entering text) without using the keyboard.

Accessibility features in the current version include the ability to:

  • Navigate and edit in Alternate, Link, Structure, and Table of Contents views of a document, which are all synchronized with the default Formatted view. For more information, see Working with Document Views.

  • Edit in the Source view, and manually synchronize it with other views. For more information, see Working with Document Views.

  • Dynamically add, remove, and edit multiple style sheets, including a user style sheet. For more information, see Working with Style Sheets.

  • Assign access keys for form fields, areas, or links. For more information, see About Access Keys.

  • Use keyboard shortcuts for menu options, links, and form controls. For more information, see Selecting from the Keyboard and About Keyboard Shortcuts.
