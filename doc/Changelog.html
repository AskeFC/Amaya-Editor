CVS Amaya changes

This file lists bug fixes and new features introduced in the CVS version of Amaya since the last release (Amaya 11.1.

New Features

  • CSS: New support of z-index, opacity, fill-rule properties

Bug fixes

  • Amaya now automatically opens the structure view when an option not visible in the formated view is created
  • The Attributes panel was not updated as soon as it is opened
  • When a column of a table is selected, attributes displayed and edited in the Attributes panel concern the <col> element. The style is attached to that <col> element too
  • Template instances: The <repeat> elements were always displayed as blocks of lines
  • Template instances: It was possible to style a fixed element
  • On Windows, local annotations were not correctly loaded
  • On Mac OS X 10.5 cache files generated by the libwww were stored in the / directory
  • SVG: When there are several <group> elements, the selection is attracted by the first <group>
  • SVG: When the <svg> element is centered, some new positions were miscalculated
  • SVG: The document redisplay was stuck when a second arrow is created
  • The Code clean-up command don't remove style and lang attributes to a <div>, <table>, <span>, and img> elements.
  • CSS: Improve the management of floated boxes and margins
  • Fix redisplay problems after a scroll
