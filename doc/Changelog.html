CVS Amaya changes

This file lists bug fixes and new features introduced in the CVS version of Amaya since the last release (Amaya 9.55).

New features

  • Redesign of the user interface: new toolbars, palettes, panels, a contextual menu
  • Possibility to display whole tools on the left, or on the right, or to custumize tool by tool
  • Change the behaviour of Enter, Delete, and Backspace keys

    a) Enter deletes the selected contents and breaks the bloc at the current position

    b) Delete deletes the selected contents and moves the cursor after

    c) Backspace deletes the selected contents and moves the cursor before

    d) On/Off buttons or entries can be used to go in/out of an inline element.
    For example you click strong, enter the strong text, then click strong to close the element.

    e) Insert and Append commands must be used to insert before and after the selected structure element

  • Implementation of Amaya themes
  • Possibility to change tabs order
  • There is now an icon in each tab area to close it
  • The application doesn't close when the last tab is closed

Bug fixes

  • Set the environment variable G_SLICE="always-malloc" to prevent a crash on new Linux platforms (the crash occurs as soon as a dialog is displayed)
  • A reference to the current page now moves to the top of page
  • Amaya generated an explicit lang="en" attribute when an element of an English document is copied and pasted into a French document
  • "Preserve Lines When Pasting" didn't work correctly in MacOS X
  • The Preferences dialog didn't display correctly
  • In dialogs, buttons that clear an URI are now displayed close to the URI
  • It's now possible to delete a link with the Link dialog
  • Maps areas could be lost
  • The CSS parser now detects the BOM character
