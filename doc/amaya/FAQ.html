Is Amaya an editor or a browser?

Amaya is mainly an authoring tool with browser features.We think that a good editor needs to be also a browser, because the management of links and the publishing are important parts of the editing task and they can be well done only if the editor integrates browsing facilities.

Where can I get Amaya?

You can download Amaya distribution file in various formats, and using either ftp or http:

  • The page http://www.w3.org/Amaya/User/BinDist.html gives access to ready to use Amaya application for a set of platforms.

  • The page http://www.w3.org/Amaya/User/SourceDist.html gives access to the source code of Amaya.

Is Amaya available for MacOS platform?

Today we support Unix platforms and Windows platforms. We don't have the resources needed for supporting another platform. But the source code is available and someone else could do the port

Can I release Amaya on a CD-ROM?

Yes, and you can charge for it, given the above requirements are met.

Can I make the changes and re-release it?

Yes, we want people to experiment with and improve Amaya. It can even be used in commercial software. If you make changes for the better, we encourage you to contact us. You may not make changes and continue to call it by a trademarked term or misrepresent the origin, capabilities, or liabilities associated with its use. You may make valid assertions, such that it is based on Amaya code, or that it is compliant with a Recommended Specification of the W3C.

Why is Amaya written in C?

C has a number of advantages that fit well with our purposes. It's the most portable and the most efficient language. Amaya is also based on a pre-existing editing library, the Thotlib, which was written in C. It was natural to continue to develop the Amaya code in C.

Is Amaya fully compliant with MathML?

Yes, but the current implementation of MathML in Amaya is not complete yet. The Amaya documentation clearly indicates what is available. See section "Current limitations" in http://www.w3.org/Amaya/User/Math.html.

Can I edit tags directly?

A plain text editing mode is difficult to handle in Amaya because Amaya works on the document tree structure and not on its text form. Amaya is able to download text documents and HTML documents. But when editing a text document, the tree structure is a set of lines and when editing an HTML document the structure tree is an HTML structure. To move from one to the other Amaya has to rebuild the whole structure tree.

Make complains with missing files

You certainly miss to create the object subtree. As explained in Autoconf page, you have to create first a subtree (obj or any other name) for object and lauch configure to create the Makefile(s)

cd Thot
mkdir obj 
cd obj
 ../configure

Make fails when compiling the HTML schema

Thot compilers need a cpp preprocessor:

  • If you use GNU tools, configure should invoque the gcc compiler with right options. You can check that the file Options has the following definition: 
    CPP     = gcc -E -x c
  • In other case you have to be sure that cpp prepocessor is in your PATH.

I don't have Motif

Motif is not free available on Linux platforms.Amaya works fine with latests versions (0.8.0 and later) of Lesstif (a free Motif clone).
