How to install the Amaya Source Release

The Amaya Source Tree

Below is the picture of the whole tree as stored in the main source archive amaya-src-xxx.tar.gz(where xxx is the Amaya version number).

One will also need to extract the precompiled libraries set archive for the Targets of interest (see the platform list) named amaya-libs-Target-xxx.tar.gz , if available (otherwise see the document on porting to a new platform).

The tree is created by invoking the following commands:

tar xvzf amaya-src-xxx.tar.gz

tar xvzf amaya-libs-Target-xxx.tar.gz

or

gunzip -c amaya-src-xxx.tar.gz | tar xvf -

gunzip -c amaya-libs-Target-xxx.tar.gz | tar xvf -

(if you are not using GNU tar) in the directory of choice (it will create the Amaya sub-directory).

Be sure to have plenty of room available, compiling a debug version of amaya can eat up to 150 Mbyte of disk space.

The tree is made of (from top to bottom):

  • amaya directory contains the amaya application code.
  • thotlib directory contains the Thot library sources. The headers defining the interfaces exported by the library are in the include sub-directory.
  • makes subdirectory, it contains most of the Makefiles and dependancies needed to rebuild amaya and the compilers from the sources.
  • batch directory contains the sources for the schema compilers.
  • font directory contains a few X-Windows specific fonts needed by amaya.
  • dicopar directory hold the dictionnaries and hyphenation rules.
  • config contains all the dialogs messages files both for english and french, as well as a few configuration files like the Thot registry (stored in file thot.ini).
  • doc contains the programming documentation for Thot library and languages.
  • libjpeg and libpng contains the sources of two graphic libraries needed by Thot toolkit.
  • w3c-libwww-5.0a is a slightly patched version of the latest version of the WWW Library.
  • LINUX-ELF, solaris2, etc.. are the Target specific files. Here is a list of all current Targets.

Each Target specific archive amaya-libs-Target-xxx.tar.gz contains a customized Makefile, precompiled versions of the graphic and WWW libraries, as well as the config.h file describing the specific issues on the programming interface of the given Target. These files are stored in a specific subdirectory Amaya/Target. This directory is also the base for the build tree related to each Target. See next step Compiling Amaya for more information on how to populate this subtree.
