How to install the Amaya-Java Binary Release

In the following, we assume that the path stored in the archive begins with the Thot directory . Currently the binary release is availabble only for a limited

Choose a place for Amaya-Java

Supposing the binary distribution was loaded fine, one must choose a place to install it. We suggest

  • /usr/local for a system wide installation
  • the user home directory for a personnal installation

An Amaya-Java binary distribution uses around 20 Megabytes of disk space, the Thot/classes/classes.zip file being responsible for half of the disk usage..

Extract the distribution

Move the distribution to this directory, check that the rights on this directory allow you to create a directory and extract the distribution.

On a Linux i386 platform one must invoke:

tar xvzf amaya-java-LINUX-Release.tar.gz

Where Release has to be replaced by the current verion number.

More generally if the distribution is a tar.gz file:

gunzip -c amaya-java-Platform-Release.tar.gz | tar xvf -

Whatever the method used, this should print list the files extracted and create a new directory called Thot populated with a few files and directories. If not, your distribution file was probably corrupted during transfer, please get a new one. Read carefully the Thot/amaya/COPYRIGHT file.

Set up the environment and start Amaya

The file Thot/SYSTEM/bin/amaya is the Amaya-Java binary, the best thing is to add the path to the Thot/SYSTEM/bin directory to your PATH environment variable. For example on a Linux i386 system:

setenv PATH $PATH:/usr/local/Thot/LINUX/bin

for a csh or tcsh shell. When using sh, bash or another variant of the Bourne shell

PATH=$PATH:/usr/local/Thot/LINUX/bin ; export PATH

Maybe this command should be added to the .login startup script. Once done one can lauch Amaya-Java simply by typing "amaya" to the shell prompt. The Amaya-Java main window should open and display the first page of the Amaya-Java documentation.

The classes used by Amaya-Java are stored in the Thot/classes directory. You don't need to add these to your CLASSPATH environment, this is done automatically when starting the Java engine. The sources for the java classes are available in the Thot/classes/sources.zip ZIP file.

If something goes wrong

  • If Amaya doesn't seems to work, check the following points:
    1. the DISPLAY variable is correctly set.
    2. the Thot/classes/ directory contains valid ZIP files
    3. eventually remove your previous registry file $HOME/.thotrc.
    4. The binary just installed is the correct one, i.e. it is the correct binaries for the current machine (don't try to start the Linux-ELF version on a Solaris machine or an old Linux a.out installation).
    5. If the binary version is ok, check the availability of the dynamic libraries needed by the executable file amayamain (located under Thot/bin/SYSTEM/amaya). for example on a Linux-ELF system type

      ldd Thot/LINUX-ELF/bin/amaya

      and check that there is no missing libraries.

    6. to debug network problems set

      DEBUG_JAVA_NET=yes

      in the registry file $HOME/.thotrc under the [amaya] section.

  • One can install packages for different platform under the same directory (for example on a server filesystem exported via NFS to an heterogenous set of machines), use the appropriate PATH on each platform.
  • All the traffic on the Amaya mailing-list is archived here. Please have a look at it before sending us mail if something goes wrong. Remember, it's an Alpha release.
