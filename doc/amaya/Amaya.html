Amaya is a Web client that acts both as a browser and as an authoring tool. It has been designed with the primary purpose of being a testbed for experimenting and demonstrating new specifications and extensions of Web protocols and standards.

Its main features include WYSIWYG editing for HTML documents including MathML elements, access to remote documents and publishing on HTTP servers, support and editing for Cascading Style Sheets. You can find a more detailed description of Amaya and of its features in the W3C Note "An Introduction to Amaya."

 Overview

Background

Amaya is built on top of the Thot library developed at Inria by project Opera. Thot offers a comprehensive toolset for structured document manipulations.

Irène Vatton and Vincent Quint started developing a HTML editor at Inria in October 1994 in order to evaluate the advantages of structured editing for authoring Web documents. This first prototype was then known as Tamaya. In October 1995, Daniel Veillard joined the team for implementing the new CSS proposal. The first demo of Tamaya was presented at the WWW4 Conference, in Boston. In February 1996, the team joined the W3C for developing Amaya, the W3C's testbed client software, based on this background.

Status

Still under development. The first public demo of Amaya took place during the Fifth International WWW Conference, in Paris (May 6-10, 1996). Source code for the current version is available for downloading.

Platforms

We have succesfully compiled the source code of Amaya on various Unix platforms and on PC/Windows.

Main Features

The main features of Amaya are as follows:

WYSIWYG editing
A document being edited is displayed with the same appearance as in a browser. The user interacts with Amaya in the same way as with a word processor.
Structure editing
The structure of a document may be displayed in a separate window. The user can clearly see the type of elements, their attributes and their hierarchy and can manipulate that structure with powerful commands.
Browsing
behaves also like a browser, with the usual user interface. Browsing and editing are integrated seamlessly.
HTML
supports the whole HTML 3.2 specification, including tables, forms, divisions, client side image maps, etc.

In the latest version, some extensions from HTML 4.0 are implemented.

Cascading Style Sheets
Document are displayed according to the style specification they contain (STYLE elements in the HEAD) and to the style sheets they refer to (LINK REL=STYLESHEET). Style elements and style sheets can be edited with Amaya.
MathML
Amaya supports a subset of the MathML specification, which makes possible to browse and edit web pages containing mathematical expressions.
HTTP
Access to remote servers is handled either by the latest version of the W3C sample HTTP library (libwww 5.0a), using either HTTP/1.0 or HTTP/1.1 or by the Java implementation of HTTP from Jigsaw. The later version is still experimental but it's already distributed in binary version.
Publishing
The PUT method of HTTP is used to write documents on remote servers. Links are updated according to the location of the document in the server.
Graphic formats
Support for pixmaps, Gif, Jpeg, PNG, ...
Printing
Generation of PostScript files. Amaya offers some facilities like printing the table of contents of the document.
Extensibility
The document structure (HTML) and the graphical aspect of documents are specified outside with specific languages allowing easy testing of new features.
API
The editor offers an API for adding new functions or for modifying existing functions.

This API is available in two flavors: C and Java.
