How to install the Amaya Binary Release

In the following, we assume that the path stored in the archive begins with the Thot directory (this change was introduced for the source distribution), however older binary distribution uses Amaya as the top directory name instead.

Choose a place for Amaya

Supposing the binary distribution was loaded fine, one must choose a place to install it. We suggest

  • /usr/local for a system wide installation
  • the user home directory for a personnal installation

A Linux version doesn't need more that 5 Megabytes of disk space, even a RISC architecture distribution should not exceed 10 Megabytes.

Extract the distribution

Move the distribution to this directory, verify the rights on this directory allows you to create a directory and extract the distribution.

On a Linux ELF platform one must invoke:

tar xvzf amaya-linux-elf.tar.gz

And for an Linux a.out platform:

tar xvzf amaya-linux.tar.gz

More generally if the distribution is a tar.gz file:

gunzip -c amaya-system.tar.gz | tar xvf -

If this is a tar.Z file:

uncompress -c amaya-system.tar.gz | tar xvf -

Whatever the method used, this should print around 40 lines of output and create a new directory called Amaya populated with a few files and directories. If not, your distribution file was probably corrupted during the transfer, please get a new one. Read carefully the COPYRIGHT and README files.

Set up the environment and start Amaya

NOTE : This has changed for Amaya 0.9. The binary is lauched directly without an intermediate script shell.

The file Thot/bin/amaya is the starting script used to launch Amaya, the best thing is to add the path to the Thot/bin/SYSTEM directory to your PATH environment variable. For example on a Linux-ELF system:

setenv PATH $PATH:/usr/local/Thot/bin/LINUX-ELF

for a csh or tcsh shell. When using sh, bash or another variant of the Bourne shell

PATH=$PATH:/usr/local/Thot/bin/LINUX-ELF ; export $PATH

Maybe this command should be added to the .login startup script. Once done one can lauch amaya simply by typing "amaya" to the shell prompt. The Amaya main window should open and display the first page of the Amaya documentation.

If something goes wrong and advice

  • If Amaya doesn't seems to work, verify the few following points:
    1. the DISPLAY variable is correctly set.
    2. The binary just installed is the correct one, i.e. it is the correct binaries for the current machine (don't try to start the Linux-ELF version on a Solaris machine or an old Linux a.out installation).
    3. If the binary version is ok, check the availability of the dynamic libraries needed by the executable file amayamain (located under Thot/bin/SYSTEM/amaya). for example on a Linux-ELF system type

      ldd Thot/bin/LINUX-ELF/amaya

      and check that there is no missing libraries.

  • One can install packages for different platform under the same directory (for example on a server filesystem exported via NFS to an heterogenous set of machines).
  • Do not try to put a symbolic link directly to the Thot/bin/amaya script from /usr/local/bin. The script would not be abble to guess correctly the value of the internal variable THOTDIR and the binary would not start.
  • The same apply when trying to copy the script to a different location.
  • All the traffic on the Amaya mailing-list is archived here. Please have a look at it before sending us mail if something goes wrong. Remember, it's an Alpha release.
