Amaya Plan

The current development phase of Amaya is divided into several activities:

  • Unix and Windows usability
  • New functionalities on HTML, XML, CSS and I18N
  • Java-DOM support

Milestones/Commitments

Future plans as of 12 January 1999:

  • 18 January, Specification draft "How to support Unicode in Amaya"
  • 22 January, Specification of "How to develop a Configuration Menu in Amaya"
  • 1 February, Separation of text management functions in Amaya
  • 1 February, Able to open/edit/save a CSS files
  • 10 February, First support of Undo in Amaya
  • 20 February, Configuration Menu
  • 15 February, CSS menus (list of properties and values)
  • 1 March, First step in Unicode support (display of a Unicode document)

  • A release in March including CSS support, Configuration menus and Undo

  • Next release in May should include the first support of Unicode in Amaya

At the same time we're opening our CVS base in Read-Only mode and we will continue to fix reported bugs.

Issues

List of possible future works (see also Wishes List from the Team) :

* I18N (in progress):

  • support of multibyte characters
  • experimentation with Unicode fonts

  • support of different writing directions

* CSS (in progress):

  • CSS file editing
  • extending the P language to stick with the CSS model: padding, margins, flowing, etc.

* XML:

  • parsing any XML document

  • display XML document using CSS rules

* DOM:

  • adding mouse events

  • adapting to the DOM API

  • experiments

* Useability (in progress):

  • Adding a configuration menu
